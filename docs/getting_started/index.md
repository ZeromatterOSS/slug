Welcome to Slug! This section will guide you through everything you need to
know to get started with Slug, from basic concepts to building your first
applications.

## What You'll Learn

This getting started guide covers the essential concepts and skills needed to
use Slug effectively:

- **Understanding Slug**: What it is and why you might want to use it
- **Installation**: How to set up Slug on your system
- **Core Concepts**: Essential terminology and concepts like targets, packages,
  and cells
- **Hands-on Tutorials**: Step-by-step practical examples building real
  applications

## Learning Path

Follow these documents in order for the best learning experience:

### 1. [What is Slug?](./what_is_slug.md)

Start here to understand what Slug is, its key features, and why you might want
to use it for your build system needs.

### 2. [Installing Slug](./install.md)

Get Slug installed and ready to use on your development machine.

### 3. [Tutorial: Your First Slug Application](./tutorial_first_build.md)

Build your first "Hello, World!" Rust application with Slug. Learn the basics
of creating BUCK files, defining targets, and building projects.

### 4. [Tutorial: Adding Dependencies](./tutorial_adding_dependencies.md)

Expand your knowledge by creating libraries, managing dependencies between
targets, and working with external dependencies.

### 5. [Tutorial: Adding Tests](./tutorial_adding_tests.md)

Learn how to write and run tests for your Slug projects to ensure code quality
and reliability.

### 6. [From Tutorial to Concepts](./core_concepts.md)

Dive deeper into Slug's core concepts by revisiting what you built in the
tutorials with a focus on understanding the underlying terminology and
architecture.

## Prerequisites

- Basic familiarity with command line tools

Ready to get started? Begin with [What is Slug?](./what_is_slug.md) to
understand the fundamentals, then follow the learning path above.
